# Overview

I am building a legislative review application. The idea is for the user to be able to enter search criteria to target a specific area of focus and paste in an API key for an AI of their choosing. Using this information the application will query the LegiScan API and return data matching their search. This data will then be sent to an AI and based on a prompt, analyze and summarize the LegiScan data and return a response to the user that will help them determine which bills are most relevant to them.

## Application flow

- A user visits the web application
- On the home page they are prompted to select from a few different options (see page layout) to narrow search criteria, enter an API key for an AI of their choosing, and provide some context about what exactly they are looking for
- They submit the request which hits the LegiScan API and returns data matching their search criteria
- That data is then sent to the AI of their choosing for bill text parsing and to determine which bills are most relevant by using the context provided by the user in the front end

## Page Layout

- Search criteria section - will be a mixture of dropdowns and free text inputs.
    - LegiScan search criteria:
        - State selection dropdown: There will be a state selection dropdown where they can choose any one of the 50 US states as well as an All States and US Congress option.
        - Full Text Search input: There will be a full text search field where the user can describe what they are looking for.
    - AI search criteria:
        - AI API Key: There will be a text input where the user can enter their AI API key.
        - AI Engine Dropdown: The user will be able to select from a number of supported AI engines (eg. Gemini, Claude, etc).
        - AI Model Dropdown (optional): The user will be able select the model they wish to interact with based on the AI engine they selected. Defaults to the latest model.
        - AI User Context Input: The user will be able to enter in some context about themselves and what they are looking for so that the AI engine knows how to parse the results from LegiScan and return the bills that are most relevant to the user. (Will need some kind of safeguard here?) 
- Raw data section - a table displaying the raw response from legiscan. Refer to the legiscan API documentation (https://legiscan.com/documentation/legiscan) for the return fields which should be used as columns in the table. 
- AI interpretation section - a table displaying the results from the AI ranking the legiscan responses by the most to least relevant to the user.
    - Table columns:
        - Relevance Score - 1-100 - determined by the AI taking into account the user context
        - Relevance Reason - max 2 sentence reason why it is relevant to the user, generated by AI
        - {Maybe some other AI generated columns?}
        - Title - same as raw data section
        - Description - comes from the `description` field on the LegiScan `getBill` response (short plain-text abstract of the bill).
        - Bill Text Link - directs user to bill text in legiscan - returned from legiscan response
        - Bill Link - directs user to bill summary in legiscan - returned from legiscan response
        - Status - indicates bill's current status in the path to legislation
        
    - Each row should be an accordion where when a user clicks on a row it expands and the summary column should show the full summary description of the bill (summary should be truncated with ellipsis to maintain screen space).
